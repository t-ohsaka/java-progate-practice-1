## 型

値にはいつくか種類がある。変数にどんな値の種類(データ型、型)を格納するのかを指定する。  
Javaには大きく分けて、型は2つある

- プリミティブ型
- 参照型

### プリミティブ型

プリミティブ型で定義した変数には**値**が格納される

以下のような、プリミティブ型(基本的な型)がある。

|名前|値の範囲|
|---|---|
|boolean|true または false|
|char|2バイト文字(\u0000 〜 \uffff)|
|byte|1バイト整数(-128 〜 127)|
|short|2バイト整数(-32768 〜 32767)|
|int|4バイト整数(-2147483648 〜 2147483647)|
|long|8バイト整数(-9223372036854775808 〜 9223372036854775807)|
|float|4バイト単精度浮動小数点数|
|double|8バイト単精度浮動小数点数|

### 参照型

参照で定義した変数には値ではなく、オブジェクトが格納されている場所への参照(リンク)が格納されている

`String`や「配列」や「その他のオブジェクト」

## 識別子

識別子とは、変数の名前として使える文字や数字の組み合わせのこと

識別子の規約

- 通常、英字・数字・アンダースコア(`_`)・`$`などの記号を使う
- 長さの制限はない
- キーワードは使えない。例）`class`、`return`など
- 数字で始めることはできない
- 大文字と小文字は区別される

識別子の例

```
num
tmp_str
print_hello
str2
```

## 演算

|算術演算子|説明|
|---|---|
|+|加算|
|-|減算|
|`*`|乗算|
|/|除算|
|%|剰余|

### いろいろな演算子

#### 文字列結合演算子

`+`で文字列の結合ができる

```java
System.out.println("hello" + "world!");
```

#### インクリメント演算子、デクリメント演算子

`++`がインクリメント演算子で、`--`がデクリメント演算子

```java
// 以下の2つは同じ意味
i = i + 1;
i++;

// 以下の2つは同じ意味
i = i - 1;
i--;
```

##### 前につけるか、後ろにつけるかで結果が変わる！？

例1）

```java
int a = 1;
int b;

b = a--;
```

bには1が代入される

例2）

```java
int a = 1;
int b;

b = --a;
```

bには0が代入される

例3）

```java
int a = 10;
int b = a++ + a;        // 10 + 11 となるため、bには21が代入される
```

#### 代入演算子

|代入演算子|説明|
|---|---|
|+=|加算代入|
|-=|減算代入|
|`*=`|乗算代入|
|/=|除算代入|

よく使うのは上の2つ

```java
// 以下の2つは同じ意味
a += 1;
a = a + 1;
```

#### 演算子の優先度

同じ優先度の場合、左から順に評価される

```java
// (a + b) + 3 と同じ
a + b + 3
```

代入は右から順に処理される

```java
// a = (b = 1) と同じ
a = b = 1;
```

文字列との結合を`+`で行うときの注意点

```java
System.out.println("1+2の結果は" + 1+2 + "です");   // 1+2の結果は12です
System.out.println("3*4の結果は" + 3*4 + "です");   // 3*4の結果は12です
```

`+`演算子は左から順に評価されるため、このような結果になってしまう。数値の計算を行う時は最も優先度の高い`()`演算子を使うこと

```java
System.out.println("1+2の結果は" + (1+2) + "です");   // 1+2の結果は3です
```

## 型の変換

サイズの大きい型への代入は可能(暗黙的な型変換)


```java
int i = 100;

double d = i;
```

> byte→short→int→long、float→double  
> 左から右への代入は暗黙的な型変換が行われる

サイズの小さい型への代入はキャスト(型変換)を行うこと

```java
double d = 3.14;

int i = (int)d;
```

※ int型は小数点を扱えないため、`3`になることに注意する。型変換を行うと、値が変わる可能性がある。


#### 異なる大きさの型で演算した場合

大きい方の型に自動でキャストされる

例えば、`double`と`int`で演算した場合

```java
2.0 * 3     // 6.0
2 * 2.0     // 6.0
```

#### 同じ大きさの型で演算した場合

演算の結果もその方になる

```java
1 / 4       // 0
1.0 / 4     // 0.25
```

もしくは、片方を型変換を行う

```java
int a = 1;
int b = 4;

double c = (double)a / b;   // cには0.25が代入される
```
